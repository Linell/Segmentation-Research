Board Counting Research.

Research directed towards counting boards in a bunk of lumber using various computer vision techniques. The research is being conducted for the University of Southern Mississippi's Honors College undergraduate thesis by me, Linell Bonnette.

It is currently in early stages, although it does show some progress.

Update: I am no longer actively working on this project. Feel free to check out the code for several pretty darn cool OpenCV examples, though!

Current State of Research

The focus of the research has shifted from counting the boards using solely photograph images into an approach that will utilize depth data from the Microsoft Kinect. Hopefully, this data will allow for greater difference in each board. Previous methods have encountered problems with differentiating between boards, resulting in an inaccurate count.

I am also transitioning from the stage of counting the boards to working on more of a block tally. Traditionally, an operator can block tally lumber that is all very similar in height, width, and length. However, with the Kinect, the block tally will be based on the exact width and height of the bunk itself, which should be more accurate. This is not as accurate as I would like for the system to be, but it is progress in the correct direction.

Download and Run

I have had a lot of trouble getting everything to run correctly. OpenCV's set up is different for every operating system and is not always easy. Drivers for the Kinect are either locked into Microsoft Windows or is not very well supported, in my experience. Regardless, here are the requirements to run the code:

  • OpenCV - used for the majority of the heavy lifting in the algorithmic department.
  • Kinect drivers - you can use whichever drivers you want, assuming that you are able to get the depth data and color images. I have not committed myself to a specific set of drivers yet, and will update this when I do.
